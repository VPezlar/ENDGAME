MPI TRANSPORT (critical detail)
  The cluster has InfiniBand (mlx5_0). tri_engine_complex uses OpenMPI 4.x
  (not the system OpenMPI 5.x). To avoid TCP BTL confusion across nodes,
  all PBS job scripts set:
    export OMPI_MCA_pml=ucx OMPI_MCA_osc=ucx OMPI_MCA_btl=self,vader
  This routes MPI traffic through UCX/InfiniBand. The UCX warnings
  "ibv_query_port_speed failed: Protocol not supported" in logs are benign.

  mpiexec used in jobs: $(dirname "$PYTHON")/mpiexec (conda's own OpenMPI 4.x)
  NOT the system mpiexec. This is critical — mixing system+conda MPI causes hangs.

4. CODEBASE STATE (feature/complex branch)
================================================================================

Key files:
  main.py          — driver; reads all params from env vars; rank-0 printing
  src/operators.py — distributed assembly; imag_shift → A += i*shift*B
  src/solver.py    — SLEPc eigensolver with MUMPS; returns timing dict
  src/FDq.py       — Python wrapper for Fortran FDq binary (1D stencils)
  src/FDq         — compiled Fortran binary (must exist; built with gfortran)
  setup_case.py    — case directory generator (see section 5)
  environment_complex.yml — NOT used directly; see section 2 for actual env

Env vars read by main.py:
  ENDGAME_NX, ENDGAME_NY, ENDGAME_NZ  — grid size per dimension (Ny,Nz default to Nx)
  ENDGAME_Q        — FD stencil full size (default 6)
  ENDGAME_MODES    — number of eigenmodes to extract (default 100)
  ENDGAME_NCV      — Krylov subspace size; must be > MODES (default 300)
  ENDGAME_TARGET   — target integer metric (default 43.0)
  ENDGAME_IMAG_SHIFT — imaginary shift injected into A (default 0.0 = real run)

5. SETUP_CASE.PY — HOW TO CREATE A CASE
================================================================================

Usage:
  python setup_case.py Nx P [queue] [walltime_h] [ppn] [imag_shift] [Ny] [Nz]

  Nx          — grid points per dimension (cubic if Ny,Nz omitted)
  P           — total MPI ranks (nodes = ceil(P/ppn))
  queue       — PBS queue (default: zeus_all_q)
  walltime_h  — walltime in hours (default: 6)
  ppn         — cores per node (default: 16, standard Zeus node)
  imag_shift  — imaginary shift 0.0=real, 0.5=complex test (default: 0.0)
  Ny, Nz      — optional; for anisotropic (non-cubic) grids

Examples:
  # Cubic real run, 4 nodes:
  python setup_case.py 60 64 zeus_all_q 4 16 0.0

  # Cubic complex test (like the validated run above):
  python setup_case.py 100 128 zeus_all_q 2 16 0.5

  # Anisotropic complex run:
  python setup_case.py 200 128 zeus_all_q 4 16 0.0 100 50

Then submit:
  cd cases/<tag> && qsub run.sh

Monitor:
  grep -v "ibv_query_port_speed\|UCX  WARN\|ib_iface" cases/<tag>/logs/run.log | tail -30

Case tag format:
  Cubic:       N{Nx}_q{Q}_P{P}
  Anisotropic: Nx{Nx}_Ny{Ny}_Nz{Nz}_q{Q}_P{P}
